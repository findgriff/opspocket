Getting Started with OpsPocket:
Your First SSH Connection

📅 April 2025 ✍️ OpsPocket Team ☕ 5 min read

Your VPS doesn't sleep, and neither do the incidents. OpsPocket puts a fully featured SSH terminal, one-tap quick actions, and OpenClaw control directly in your pocket — so when something breaks at 3am, you can fix it from your phone before you've finished making coffee.

What Is OpsPocket?

OpsPocket is an iPhone-native SSH console built specifically for developers, DevOps engineers, and VPS operators who run AI services, bots, and web infrastructure. It's not a generic SSH client bolted onto a phone — it was designed from the ground up for the workflows of people running OpenClaw, Clawbot instances, Docker stacks, PM2 processes, and systemd services on Linux servers.

Think of it as a command centre in your pocket: real-time SSH terminal, slash-command palette for common operations, secure credential storage using Face ID and iOS Keychain, and deep integration with OpenClaw's gateway and agent ecosystem.

💡
Who is this for? OpsPocket is built for people who are already comfortable with Linux and SSH. It's not a server setup tool — it's a server management tool for when you're away from your desk.

What Can OpsPocket Do?

Here's a quick tour of everything the app gives you out of the box:

🖥️

Full SSH Terminal

Real interactive terminal — tab completion, arrow keys, Ctrl shortcuts, ANSI colour — just like your desktop client.

⚡

Slash Commands

Type / in the terminal input and a dropdown appears with 60+ pre-built commands for Docker, PM2, systemd, tmux, logs and OpenClaw.

🔐

Face ID Protection

All SSH credentials, passwords, and private keys are stored in the iOS Keychain, unlocked with Face ID. Nothing lives in plain text.

🔑

SSH Key Auth

Paste your private key once at setup time. OpsPocket stores it in the Keychain and uses it for every connection — no passwords exposed.

🐙

OpenClaw Native

First-class support for OpenClaw commands: gateway management, skill installs, agent status, channel probes and more — all slash commands.

📋

Quick Actions

One-tap tiles for your most-used operations per server. Restart Nginx, flush Redis, check disk space — without typing a thing.

🌐

The Tunnel

SSH-tunnel any local web interface (like Portainer or your OpenClaw dashboard) and open it in an in-app browser — securely, no public port needed.

🖧

Multi-Server

Add as many servers as you need. Each server has its own credentials, quick actions, and session history — fully isolated.

Step 1 — Install OpsPocket

Download OpsPocket from the App Store and open it. On first launch, you'll be prompted to authenticate with Face ID — this sets up the secure Keychain vault where all your credentials will live. This happens once; after that, Face ID unlocks the app automatically.

🔒
Security note OpsPocket never sends your credentials to any external server. All SSH keys, passwords, and server details are stored locally in the iOS Keychain. The app communicates directly from your phone to your VPS.

Step 2 — Add Your First Server

Tap the + button on the home screen to add a new server profile. You'll need:

  • Label — a friendly name, e.g. "Production VPS"
  • Host — your server's IP address or hostname
  • Port — usually 22, but use whatever yours is configured to
  • Username — e.g. ubuntu, root, or your custom user
  • Authentication — either a password or an SSH private key (recommended)

Using SSH Key Authentication (Recommended)

If you use key-based auth (and you should), paste your private key directly into the key field during server setup. OpsPocket will move it into the Keychain immediately — it's never written to disk anywhere else. If your key has a passphrase, enter that too and it will be stored alongside.

💡
Getting your private key onto your phone The easiest way: open your terminal on your Mac, copy your key with pbcopy < ~/.ssh/id_ed25519, AirDrop a text file to your iPhone, paste it into OpsPocket. Delete the file after — the Keychain is the only place it needs to live.

Step 3 — Open the Terminal

Tap your newly added server card. OpsPocket will connect via SSH and open the interactive terminal session. The connection is immediate — you'll see your server's shell prompt within a second or two on a good connection.

The terminal supports:

  • Full ANSI colour output — htop, git log --oneline, coloured logs
  • Arrow key navigation — use up/down for command history
  • Ctrl shortcuts — Ctrl+C, Ctrl+Z, Ctrl+L all work as expected
  • Landscape mode — rotate your phone for a wider terminal view
  • Auto-scroll with a tap on the latest output

Step 4 — Use Slash Commands

This is where OpsPocket goes beyond a generic SSH client. Tap the terminal input and type / — a dropdown appears immediately with all available slash commands, filtered as you type.

# Type / to open the palette, then filter by keyword /docker → shows all Docker slash commands /logs → shows log tail commands /restart → all restart commands (pm2, systemd, docker) /openclaw → all OpenClaw management commands

Tap any command and it populates the input field. If the command has a placeholder (like a container name or service name), you'll be prompted to fill it in before it sends. Press Send and it runs on your server.

Categories in the Slash Palette

  • Status — disk usage, uptime, CPU/memory, active connections
  • Logs — tail logs, journalctl, docker logs, follow mode
  • Restart — restart services, containers, processes
  • Docker — ps, stats, exec, compose up/down, image prune
  • PM2 — list, restart, logs, save, startup
  • systemd — status, start, stop, restart, enable, journalctl
  • tmux — list sessions, attach, kill
  • OpenClaw — gateway, agents, skills, cron, models, config, backup
  • Custom — commands you've saved yourself

Step 5 — Quick Actions

In the server detail screen, you can configure Quick Actions — tiles that execute a single command with one tap, no typing required. These are perfect for the operations you run most often on a specific server.

Some examples to add as Quick Actions:

# Some great first Quick Actions to set up: df -h # Disk usage at a glance docker ps --format "table ..." # Running containers pm2 list # PM2 process health openclaw status # OpenClaw gateway state systemctl --user status openclaw-gateway # Gateway systemd unit

Step 6 — Save Custom Commands

Any command you find yourself running repeatedly can be saved as a custom slash command. From the terminal screen, long-press a command you've typed or tap the bookmark icon to save it. Give it a slash alias (e.g. /mybackup) and a label, and it'll appear in your personal section of the slash palette.

You can also add placeholders in double braces — e.g. docker restart {{container_name}} — and OpsPocket will prompt you to fill the blank before running.

What Happens When Your Session Disconnects?

Mobile SSH sessions drop — it's a fact of life. When your iPhone goes to sleep, switches networks, or iOS decides to reclaim background memory, the SSH connection will close. OpsPocket handles this gracefully:

  • The terminal shows a clear "Disconnected" state — no silent confusion
  • A single tap reconnects immediately using the stored credentials
  • Your session history (scroll buffer) is preserved so you can review what happened
⚠️
Long-running commands If you need to run something that takes a while (a build, a migration, a model download), always start it inside a tmux or screen session. That way, if the SSH connection drops, the process continues on the server. Use /tmux from the slash palette.

Tips & Tricks for Daily Use

1

Use landscape mode for log tailing

Rotate your phone when reading logs or working in htop — you get roughly double the terminal width, which makes dense output much more readable.

2

Pin your most-used server at the top

Long-press a server card on the home screen to reorder. Keep your most-critical server at the top for one-tap access in an emergency.

3

Build a Quick Actions emergency kit

For every server, set up at least: check disk, restart the main service, tail the error log. These three get you through 80% of 3am incidents.

4

Use OpenClaw slash commands if you run AI services

Type /openclaw to see the full suite — gateway restart, skill management, agent status, cron jobs. It's a full management layer without needing to remember every command.

5

The Tunnel for web interfaces

If you run Portainer, Grafana, or an OpenClaw web UI on a local port, set up a Tunnel in the server detail screen. OpsPocket creates an SSH tunnel and opens it in the in-app browser — no public exposure needed.
